待办事项

在线Markdown编辑器

做这个博客是为了巩固 web 开发的相关知识。后台、前端都从零开始。

所以先是自己写了个 web 框架，基本功能完成后，迁移到了 flask。flask 简洁，灵活。

用户管理、session / Cookie、csrf 防护、ajax、json 接口等从头做起。

部署到服务器时数据库选择了mongoDB。

接下来就作为学习笔记本吧。

待办事项是最简单的一个 web 应用，涵盖的知识点又基本覆盖了 web 开发的所有相关知识，所以搞懂了待办事项，基本就可以应对基础的 web 开发了。

待办事项最初用 Python 写后端接口，前端静态页面显示；后来使用原生 javascript 和 chrome 的 localstorage 重写了一遍；接着又用 jQuery 写了一遍； 最后接口又改成了 ajax 。全部练习了一遍。

下一步的计划是改写待办事项，页面和功能参照 google keep，用网页实现一个简洁版的 google keep。

在线 Markdown 编辑器，目前做好的主要功能有实时预览，同步预览，基本的样式，接下来计划加上代码高亮的功能。

编辑器模块可以重复使用，目前博客的文章编辑和修改就是用的这个编辑器模型。

最终要把这个模块做成一个功能完善的笔记本应用。
